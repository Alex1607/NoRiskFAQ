Wie kann ich den Client aktuell benutzen?
Der Client befindet sich in der Open Beta. 
Man kann ihn also bereits auf norisk.gg herunterladen und starten aber damit man mit dem Client spielen kann, muss man auf der Whitelist stehen.

Wie kommt man auf die Whitelist?
Nein, man kann sich nicht bewerben
Nein, man kann sich nicht auf die Whitelist kaufen
Nein, man kann sich nicht durch ein Ticket für die Whitelist bewerben

Um auf die Whitelist zu kommen musst du den Channel <#1258102216854732820> nutzen.
In diesem Channel musst du über den Button, welcher sich nur aktiviert wenn neue Beta User gebraucht werden, der Beta beitreten.
Dies wird über EINEN oder MEHRERE der folgenden Wege kommuniziert:

Im Video von Norisk auf https://www.youtube.com/@noriskk
Im Whatsapp Channel von Norisk https://www.whatsapp.com/channel/0029VaEHVubEgGfRBa5pzW29
Hier auf dem Discord in <#774271756549619725>

Dann kannst du auf den Button drücken, deinen Minecraft Namen eingeben und den benötigten Beta Code. Es kann sein das ein bestimmter Code gebraucht wird um sich in die Whitelist zu schreiben, dieser wird kommuniziert und der Code MUSS in das dafür vorgesehene Feld eingetragen werden, sonst funktioniert es nicht.

Wie kann ich die Version ändern?
Zurzeit kannst du nur mit der neuste Version spielen, es ist nicht möglich die 1.8 oder andere Versionen auszuwählen.

Wo finde ich das NRC Emote base Model?
"Geckolib Animation Utils" Plugin in Blockbench installieren.
Neues "Geckolib Animated Model" erstellen und beim Erstellen als "Model Type" -> "Armor" auswählen.

Alternativ:
Im <#774292827524956181> Channel könnt ihr euch das Model herunterladen.
(Das Plugin braucht ihr trotzdem!)

Achtung:
Die Pivot Points (Ankerpunkte) dürfen nicht verschoben werden. Lasst einfach das Pivot Tool in Ruhe.

Beim Installieren bekomme folgende Meldung: Der Computer wurde durch Windows geschützt
Das liegt daran, dass wir noch kein Zertifikat haben, weil das teuer ist.
Um diesen Schutz zu umgehen, klicke auf "Weitere Informationen" und "Trotzdem ausführen".
Wenn du den Client von https://norisk.gg/ heruntergeladen hast, ist er sicher.

MacOS
Wenn ihr auf macOS diese Fehlermeldung beim Öffnen bekommt, müsst ihr auf "Abbrechen", anschließend in eure Einstellungen auf  "Datenschutz & Sicherheit" und dann relativ weit unten unter dem Punkt "Sicherheit" auf "Trotzdem öffnen". Dann euer Passwort bzw. Fingerabdruck bestätigen und den Launcher neu öffnen.

Wie lange dauert mein Cape angenommen wird und woher weiß ich ob es angenommen wurde oder nicht?
Wenn es angenommen wurde findest du es in deinem cape menu im Launcher, die Zeit bis es angenommen wird kann sich unterscheiden sollte aber maximal einige Stunden dauern.

Mein Cape wurde nicht angenommen, was kann der Grund dafür sein?
Es hat potenziell gegen die Cape-Regeln verstoßen, diese sind:
Keine urheberrechtlich geschützten Inhalte (z.B. Umhänge von anderen Kunden, Inhalte im Zusammenhang mit Nintendo usw.) -> das Umfärben eines urheberrechtlich geschützten Bildes entfernt das Copyright nicht 
Keine expliziten Inhalte -> Nacktheit etc.
Keine diskriminierenden, rassistischen usw. Inhalte
Keine politischen Inhalte (Igbtq-Flaggen sind nicht politisch, bitte werdet erwachsen)
oder, was häufig auch der Fall ist, dass die Elytra vergessen wurde. Diese ist ein Teil deines Capes und muss immer auch gefüllt sein.

Launcher startet nicht?
Start im privileged mode (run as admin)
Taskkill im Task Manager oder Commandline
3rd-Party Services deaktivieren (Antivirus, Firewall)
Advanced (Windows only): CMD -> als Admin ausführen -> netsh winsock reset (bei bootloop of death)

Launcher nur weiß/schwarz (leer)?
Neuinstallieren
Keine Dateien verändern
Launcher-Ordner löschen
Windows: %appdata%\\norisk\\NoRiskClient\\config
Linux: Ctrl + H (versteckte Dateien anzeigen) -> /home/[Benutzer]/.config/noriskclient/options.json
Mac: Cmd + Shift + . (versteckte Dateien anzeigen) -> Benutzer\\[Benutzer]\\Library\\ApplicationSupport\\gg.norisk.NoRiskClient\\options.json

Wie nutze ich den Client bei Linux?
"Unacceptable TLS Certificate" bei Login: $ sudo ln -s /etc/ssl/ca-bundle.pem /etc/ssl/certs/ca-certificates.crt
Launcher UI cursed: $ export WEBKIT_DISABLE_COMPOSITING_MODE=1
AppImage is AMD only, AARCH/ARM Anleitung in Readme)
Troubleshooting AppImage/.deb:
sudo apt update -y
sudo apt upgrade -y
sudo apt install libfuse2 (libfuse2t64 for Ubuntu)

Debian: libgtk-3-dev libssl-dev libsoup2.4-dev libwebkit2gtk-4.0-dev
Arch: cairo gdk-pixbuf2 glib2 gtk3 libsoup openssl pango webkit2gtk
